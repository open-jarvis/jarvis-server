HTTP API

For every request, you have to provide a pre-shared key in the post body: {"psk":"XXXX"}
Requests without the pre-shared keys or with invalid pre-shared keys are automatically cancelled!
/generate-token
Generates a token
Generates a one-time token for your application. This token is valid for 120 seconds.
/register-device GET Parameters name Your device name token A previously generated token (must not be older than 120 seconds) type Your device type (supported values: computer, laptop, tablet or mobile). You can use a custom device type too native If your app is connected through a native app ("true") or a web-app ("false")
Registers a device
Registers a device on this server. This device is then allowed to send status updates and infos
/unregister-device GET Parameters token The same token you used to register your device
Unregisters a device
Unregisters a device on this server. Use this endpoint only when you don't want your device to share information anymore.
Do not use this endpoint, if your device is offline
/set-property GET Parameters token The same token you used to register your device POST Parameters property The device property to set value The device property's value to set
Set a device property
Sets a device property name and value. Recommended properties are notifications, location, battery, etc...
value has to be a dictionary (default in get is {} )
/get-property GET Parameters token Your device token POST Parameters property The device property to get
Get a device property
Get a device property by name. If no token is set, get property from all devices
/hello GET Parameters token Your device token
Mark device as online
Make sure to send a /hello request every 10-15 seconds. After 20s of inactivity, your device will be marked as offline.
/am-i-registered GET Parameters token Your device token
Check if device still exists
Use this endpoint, after your application closed and gets reopened. This will check, if your token is still valid, or you'll need to generate a new one!
/start-debug GET Parameters token Your device token
Start a debugging session
Starts a debugging session, use the /scan-debug endpoint to get new updates
/scan-debug GET Parameters token Your device token (the token you called /start-debug with)
Scan for debugging updates
Returns all new updates since the last /scan-debug call.
You can call this endpoint quite frequently, as it doesn't touches the file system but stores data in RAM (every 0.5s is fine)
/end-debug GET Parameters token Your device token
End a debugging session
Ends your debugging session and clears all data

InstantDecision API

The ID API is used to request real-time decisions from other services
/id/ask GET Parameters token Your device token POST Parameters type Decision type to make. This is a unique identifier name Display name of the decision to make infos String providing all informations to make this decision. options Options object
Ask for a decision
Asks for an instant decision of "type" with a display name.
/id/answer GET Parameters token Your device token POST Parameters target_token Target device token type Decision type to make. This is a unique identifier option Index in the options object description (optional) String providing additional information for this decision.
Answer a request
Answers a /aks request.
/id/scan GET Parameters token Your device token POST Parameters target_token (optional) Target device token type (optional) Scan for a given type
Scan for requests
Scan for requests.

If "token" is set, it only scans for requests from the given device token (use this to scan for answers).
If "type" is set, only return requests with the given type (use this to scan for ask requests).
/id/delete GET Parameters token Your device token POST Parameters target_token Target device token type The type to delete
Delete a decision
Deletes a decision. Call this API endpoint either if the decision wasn't anwered on time or got answered and you don't need the decision storage anymore
